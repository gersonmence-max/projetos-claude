O QUE É
-----------------------------------------------------------------------------

Bot Telegram para gestão operacional da Ziontec Power Solutions. Gerencia
projetos, clientes, ordens de serviço, follow-ups, e integração com Google
Sheets como banco de dados. Sistema determinístico com fallback para LLMs
(Claude Haiku, Groq, Gemini).

OBJETIVO ESTRATÉGICO: Não é só um bot — é o **piloto do SMB OS**. A lógica
de gestão aqui valida o que vai virar SaaS para outras empresas.

-----------------------------------------------------------------------------
DECISÕES ARQUITETURAIS (ADRs)
-----------------------------------------------------------------------------

Telegram em vez de WhatsApp:
- API gratuita e ilimitada
- Bots nativos (não precisa Business API)
- Interface mais flexível (botões inline, etc)

Google Sheets como MVP DB:
- Zero setup
- Fácil de debugar (vê dados em tempo real)
- Suficiente para single-tenant
- Migração para PostgreSQL quando multi-tenant

Lógica determinística + LLM fallback:
- Comandos estruturados são mais rápidos e baratos
- LLM só para casos ambíguos
- Custo operacional menor

Claude Haiku como LLM primário:
- Melhor custo-benefício ($0.25/1M tokens)
- Latência baixa (~1s)
- Qualidade suficiente para assistente

Separação bot.py / core.py:
- Core testável independentemente
- Fácil migrar para web (reusa core.py)
- Melhor manutenibilidade
