-----------------------------------------------------------------------------
O QUE É
-----------------------------------------------------------------------------

Sistema de análise de viabilidade para instalação de carregadores de veículos
elétricos (DCFC e Level 2) em Massachusetts. Pipeline de 5 agentes que avalia
demanda, competição, adequação do local e afinidade com EVs para gerar score
de oportunidade de negócio.

Objetivo: Identificar as melhores cidades de Massachusetts para instalar
carregadores EV e gerar leads para a Ziontec.

-----------------------------------------------------------------------------
STACK TÉCNICA
-----------------------------------------------------------------------------

Backend:  Python + Flask (porta 5000)
Frontend: React + Vite + TailwindCSS (porta 5173)
APIs:     LocationIQ (geocoding)
          TomTom (tráfego)
          OCM - Open Charge Map (chargers existentes)
          Google Places (pontos de interesse)
Banco:    SQLite
Cidades:  341 municípios de Massachusetts
Infra:    VPS Hetzner (planejado)

-----------------------------------------------------------------------------
PIPELINE DE 5 AGENTES
-----------------------------------------------------------------------------

1. GEOCODING AGENT
   - LocationIQ → OpenCage → Nominatim (fallback)
   - Obtém coordenadas da cidade

2. LOCAL SEARCH AGENT
   - Google Places API
   - Identifica pontos de interesse (restaurantes, hotéis, shopping centers)

3. VIABILITY AGENT
   - TomTom Traffic API (densidade de tráfego)
   - OCM (chargers já existentes na área)
   - Cruza dados de demanda vs competição

4. SCORING AGENT
   - Demand (peso 35%): Tráfego + POIs de alto valor
   - Competition (peso 25%): Chargers existentes / densidade
   - Site Fit (peso 20%): Adequação do local (zoning, acesso)
   - EV Affinity (peso 20%): Renda média, educação, adoção de EVs

5. REPORTING AGENT
   - Gera relatório consolidado
   - Ranking de cidades
   - Exportação PDF (planejado)

-----------------------------------------------------------------------------
DECISÕES ARQUITETURAIS (ADRs)
-----------------------------------------------------------------------------

Flask em vez de FastAPI:
- Projeto iniciado antes do Tax Deed Finder
- Flask é mais simples para MVPs rápidos
- Migração para FastAPI pode ser feita depois

SQLite em vez de PostgreSQL:
- Single-user, sem multi-tenancy
- 341 cidades é dataset pequeno
- Simplicidade de deploy

Pipeline de agentes sequencial:
- Mais fácil de debugar que paralelo
- Latência não é crítica (batch processing)
- Clareza de lógica

Massachusetts como estado piloto:
- Ziontec já opera em MA
- Incentivos estaduais para EVs
- Alta densidade populacional
