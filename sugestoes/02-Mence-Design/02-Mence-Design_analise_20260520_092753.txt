### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas. Por exemplo, em vez de:
```sql
SELECT * FROM usuarios WHERE nome = '$nome';
```
Utilizar:
```sql
SELECT * FROM usuarios WHERE nome = @nome;
```
E passar o parâmetro `@nome` separadamente.

**Exemplo de arquivo:** `database.php` ou `models.py`

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações críticas, como atualizações de estoque ou processamento de pagamentos, podem falhar parcialmente, deixando o sistema em um estado inconsistente.

**Solução:** Utilizar transações para garantir que as operações sejam executadas de forma atômica. Por exemplo:
```sql
BEGIN TRANSACTION;
UPDATE estoque SET quantidade = quantidade - 1 WHERE id = @id;
INSERT INTO pedidos (id_usuario, id_produto) VALUES (@id_usuario, @id_produto);
COMMIT;
```
**Exemplo de arquivo:** `orders.php` ou `payment.py`

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Consultas podem ser lentas devido à falta de índices em foreign keys e campos de busca.

**Solução:** Criar índices em foreign keys e campos de busca. Por exemplo:
```sql
CREATE INDEX idx_id_usuario ON pedidos (id_usuario);
CREATE INDEX idx_nome ON produtos (nome);
```
**Exemplo de arquivo:** `database.php` ou `models.py`

### 4. Uso de SELECT * Desnecessário

**Problema:** Consultas podem retornar mais dados do que o necessário, aumentando o tráfego de rede e a carga no banco de dados.

**Solução:** Especificar apenas as colunas necessárias nas consultas. Por exemplo:
```sql
SELECT id, nome, preco FROM produtos WHERE id = @id;
```
**Exemplo de arquivo:** `products.php` ou `product.py`
