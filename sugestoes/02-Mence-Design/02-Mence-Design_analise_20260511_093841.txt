### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas em vez de concatenadas. Por exemplo, em vez de:
```sql
SELECT * FROM usuarios WHERE nome = '" + nome + "' AND senha = '" + senha + "'";
```
Utilizar:
```sql
SELECT * FROM usuarios WHERE nome = @nome AND senha = @senha;
```
E passar os parâmetros `@nome` e `@senha` separadamente.

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Falta de transações pode levar a inconsistências nos dados em caso de falhas.

**Solução:** Utilizar transações para garantir a integridade dos dados. Por exemplo:
```sql
BEGIN TRANSACTION;
INSERT INTO pedidos (id, cliente_id, total) VALUES (@id, @cliente_id, @total);
INSERT INTO itens_pedido (pedido_id, produto_id, quantidade) VALUES (@id, @produto_id, @quantidade);
COMMIT TRANSACTION;
```
### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Falta de índices pode levar a consultas lentas e ineficientes.

**Solução:** Criar índices em foreign keys e campos de busca. Por exemplo:
```sql
CREATE INDEX idx_cliente_id ON pedidos (cliente_id);
CREATE INDEX idx_produto_id ON itens_pedido (produto_id);
```
### 4. Uso de SELECT * Desnecessário

**Problema:** Uso de `SELECT *` pode retornar mais dados do que necessário, afetando o desempenho.

**Solução:** Especificar apenas as colunas necessárias. Por exemplo:
```sql
SELECT id, nome, email FROM clientes;
```

Em vez de:
```sql
SELECT * FROM clientes;
```
