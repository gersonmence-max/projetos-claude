### 1. SQL Injection (queries concatenadas vs parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas.

**Exemplo:**

```sql
-- Problema
SELECT * FROM usuarios WHERE nome = '$nome';

-- Solução
SELECT * FROM usuarios WHERE nome = @nome;
```

### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como atualizações de estoque ou processamento de pagamentos, podem ser afetadas por falhas de concorrência ou erros.

**Solução:** Utilizar transações para garantir a integridade dos dados.

**Exemplo:**

```sql
-- Problema
UPDATE estoque SET quantidade = quantidade - 1 WHERE produto_id = 1;
INSERT INTO pedidos (produto_id, quantidade) VALUES (1, 1);

-- Solução
BEGIN TRANSACTION;
UPDATE estoque SET quantidade = quantidade - 1 WHERE produto_id = 1;
INSERT INTO pedidos (produto_id, quantidade) VALUES (1, 1);
COMMIT;
```

### 3. Falta de índices em foreign keys e campos de busca

**Problema:** Consultas podem ser lentas devido à falta de índices.

**Solução:** Criar índices em foreign keys e campos de busca.

**Exemplo:**

```sql
-- Problema
CREATE TABLE pedidos (
    id INT PRIMARY KEY,
    produto_id INT,
    FOREIGN KEY (produto_id) REFERENCES produtos(id)
);

-- Solução
CREATE TABLE pedidos (
    id INT PRIMARY KEY,
    produto_id INT,
    FOREIGN KEY (produto_id) REFERENCES produtos(id),
    INDEX (produto_id)
);
```

### 4. Uso de SELECT * desnecessário

**Problema:** Consultas podem retornar mais dados do que necessário, afetando o desempenho.

**Solução:** Especificar apenas as colunas necessárias.

**Exemplo:**

```sql
-- Problema
SELECT * FROM produtos;

-- Solução
SELECT id, nome, preco FROM produtos;
```
