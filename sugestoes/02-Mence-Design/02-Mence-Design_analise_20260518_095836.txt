### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas podem permitir ataques de SQL Injection, onde um atacante pode injetar código malicioso no banco de dados.

**Solução:** Utilizar queries parametrizadas, onde os parâmetros são passados separadamente da query, evitando assim a possibilidade de injeção de código malicioso.

**Exemplo:**

```sql
-- Problema
SELECT * FROM usuarios WHERE nome = '$nome';

-- Solução
SELECT * FROM usuarios WHERE nome = @nome;
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações críticas, como atualizações de estoque ou processamento de pagamentos, podem falhar parcialmente, deixando o sistema em um estado inconsistente.

**Solução:** Utilizar transações para garantir que as operações sejam executadas de forma atômica, ou seja, todas as partes da operação são executadas com sucesso ou nenhuma delas é executada.

**Exemplo:**

```sql
-- Problema
UPDATE estoque SET quantidade = quantidade - 1 WHERE id = 1;
INSERT INTO pedidos (id_usuario, id_produto) VALUES (1, 1);

-- Solução
BEGIN TRANSACTION;
UPDATE estoque SET quantidade = quantidade - 1 WHERE id = 1;
INSERT INTO pedidos (id_usuario, id_produto) VALUES (1, 1);
COMMIT;
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** A falta de índices em foreign keys e campos de busca pode levar a consultas lentas e ineficientes.

**Solução:** Criar índices em foreign keys e campos de busca para melhorar a performance das consultas.

**Exemplo:**

```sql
-- Problema
CREATE TABLE pedidos (id_usuario INT, id_produto INT);

-- Solução
CREATE TABLE pedidos (id_usuario INT, id_produto INT);
CREATE INDEX idx_id_usuario ON pedidos (id_usuario);
CREATE INDEX idx_id_produto ON pedidos (id_produto);
```

### 4. Uso de SELECT * Desnecessário

**Problema:** O uso de SELECT * pode retornar mais dados do que o necessário, o que pode afetar a performance da consulta.

**Solução:** Especificar apenas as colunas necessárias na consulta.
