### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas. Por exemplo, em vez de:
```sql
SELECT * FROM usuarios WHERE nome = '$nome';
```
Utilizar:
```sql
SELECT * FROM usuarios WHERE nome = @nome;
```
E passar o parâmetro `@nome` separadamente.

**Exemplo de arquivo:** `database.php`
**Query problemática:** `SELECT * FROM usuarios WHERE nome = '$nome';`
**Versão corrigida:** `SELECT * FROM usuarios WHERE nome = @nome;`

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações críticas, como atualizações de estoque ou processamento de pagamentos, podem falhar parcialmente, deixando o sistema em um estado inconsistente.

**Solução:** Utilizar transações para garantir que as operações sejam executadas de forma atômica. Por exemplo:
```sql
BEGIN TRANSACTION;
UPDATE estoque SET quantidade = quantidade - 1 WHERE id = @id;
INSERT INTO pedidos (id_usuario, id_produto) VALUES (@id_usuario, @id_produto);
COMMIT;
```
**Exemplo de arquivo:** `order.php`
**Query problemática:** `UPDATE estoque SET quantidade = quantidade - 1 WHERE id = @id; INSERT INTO pedidos (id_usuario, id_produto) VALUES (@id_usuario, @id_produto);`
**Versão corrigida:** `BEGIN TRANSACTION; UPDATE estoque SET quantidade = quantidade - 1 WHERE id = @id; INSERT INTO pedidos (id_usuario, id_produto) VALUES (@id_usuario, @id_produto); COMMIT;`

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Consultas que utilizam foreign keys ou campos de busca podem ser lentas devido à falta de índices.

**Solução:** Criar índices nos campos relevantes. Por exemplo:
```sql
CREATE INDEX idx_id_usuario ON pedidos (id_usuario);
CREATE INDEX idx_nome ON produtos (nome);
```
**Exemplo de arquivo:** `database.php`
**Query problemática:** `SELECT * FROM pedidos WHERE id_usuario = @id_usuario;`
**Versão corrigida:** `CREATE INDEX idx_id_usuario ON pedidos (id_usuario); SELECT * FROM pedidos WHERE id_usuario = @id_usuario;`
