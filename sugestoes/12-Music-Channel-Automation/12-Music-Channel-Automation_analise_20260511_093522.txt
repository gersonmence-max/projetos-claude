### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações críticas, como atualizações de banco de dados, podem não ser tratadas como transações, o que pode levar a inconsistências nos dados.

**Exemplo de problema:**
```python
import sqlite3

# Conexão com o banco de dados
conn = sqlite3.connect('music_channel.db')
cursor = conn.cursor()

# Atualização de dados sem transação
cursor.execute("UPDATE usuarios SET nome = 'Novo Nome' WHERE id = 1")
cursor.execute("UPDATE playlists SET nome = 'Nova Playlist' WHERE id = 1")
```

**Versão corrigida:**
```python
import sqlite3

# Conexão com o banco de dados
conn = sqlite3.connect('music_channel.db')
cursor = conn.cursor()

# Atualização de dados com transação
try:
    cursor.execute("BEGIN TRANSACTION")
    cursor.execute("UPDATE usuarios SET nome = 'Novo Nome' WHERE id = 1")
    cursor.execute("UPDATE playlists SET nome = 'Nova Playlist' WHERE id = 1")
    cursor.execute("COMMIT")
except sqlite3.Error as e:
    cursor.execute("ROLLBACK")
    print(f"Erro: {e}")
```
