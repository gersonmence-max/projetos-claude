### 1. SQL Injection

**Problema:** Queries concatenadas que permitem a injeção de código malicioso.

**Solução:** Utilizar queries parametrizadas. Por exemplo, se estiver usando Python com PostgreSQL, você pode usar o `psycopg2` com parâmetros nomeados.

**Exemplo Problemático:**
```python
cursor.execute("SELECT * FROM clientes WHERE nome = '" + nome + "'")
```

**Versão Corrigida:**
```python
cursor.execute("SELECT * FROM clientes WHERE nome = %s", (nome,))
```

### 2. Ausência de Transactions

**Problema:** Operações críticas que não são encapsuladas em transações, podendo levar a inconsistências nos dados.

**Solução:** Utilizar transações para garantir a integridade dos dados. Por exemplo, ao criar uma ordem de serviço e atualizar o status de um projeto, essas operações devem ser feitas dentro de uma transação.

**Exemplo Problemático:**
```python
# Criar ordem de serviço
cursor.execute("INSERT INTO ordens_de_servico (projeto_id, descricao) VALUES (%s, %s)", (projeto_id, descricao))
# Atualizar status do projeto
cursor.execute("UPDATE projetos SET status = 'em_andamento' WHERE id = %s", (projeto_id,))
```

**Versão Corrigida:**
```python
with cursor:
    cursor.execute("BEGIN")
    try:
        # Criar ordem de serviço
        cursor.execute("INSERT INTO ordens_de_servico (projeto_id, descricao) VALUES (%s, %s)", (projeto_id, descricao))
        # Atualizar status do projeto
        cursor.execute("UPDATE projetos SET status = 'em_andamento' WHERE id = %s", (projeto_id,))
        cursor.execute("COMMIT")
    except Exception as e:
        cursor.execute("ROLLBACK")
        raise e
```
