### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.
**Solução:** Utilizar queries parametrizadas.

**Exemplo de Problema:**
```sql

```
**Versão Corrigida:**
```sql
query = "SELECT * FROM usuarios WHERE nome = %s"
cursor.execute(query, (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Falta de transações pode levar a inconsistências nos dados.
**Solução:** Utilizar transações para garantir a integridade dos dados.

**Exemplo de Problema:**
```sql
cursor.execute("INSERT INTO pedidos (id_cliente, total) VALUES (%s, %s)", (id_cliente, total))
cursor.execute("UPDATE clientes SET saldo = saldo - %s WHERE id = %s", (total, id_cliente))
```
**Versão Corrigida:**
```sql
with connection.cursor() as cursor:
    try:
        cursor.execute("BEGIN TRANSACTION")
        cursor.execute("INSERT INTO pedidos (id_cliente, total) VALUES (%s, %s)", (id_cliente, total))
        cursor.execute("UPDATE clientes SET saldo = saldo - %s WHERE id = %s", (total, id_cliente))
        cursor.execute("COMMIT")
    except Exception as e:
        cursor.execute("ROLLBACK")
        raise e
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Falta de índices pode levar a consultas lentas.
**Solução:** Criar índices em foreign keys e campos de busca.

**Exemplo de Problema:**
```sql
CREATE TABLE pedidos (
    id INT PRIMARY KEY,
    id_cliente INT,
    total DECIMAL(10, 2)
)
```
**Versão Corrigida:**
```sql
CREATE TABLE pedidos (
    id INT PRIMARY KEY,
    id_cliente INT,
    total DECIMAL(10, 2),
    FOREIGN KEY (id_cliente) REFERENCES clientes (id),
    INDEX idx_id_cliente (id_cliente)
)
```
