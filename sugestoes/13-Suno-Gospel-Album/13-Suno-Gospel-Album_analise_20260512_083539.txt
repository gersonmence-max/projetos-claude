**Algoritmos de assinatura JWT (HS256 vs RS256)**

* HS256 é um algoritmo de assinatura simétrica, que utiliza uma chave secreta compartilhada entre o emissor e o verificador do token. É mais rápido e eficiente, mas pode ser mais vulnerável a ataques de força bruta.
* RS256 é um algoritmo de assinatura assimétrica, que utiliza um par de chaves (pública e privada). É mais seguro, pois a chave privada é utilizada apenas para assinar o token, e a chave pública é utilizada para verificar a assinatura.

**Expiração e rotação de tokens**

* É importante definir um tempo de expiração para os tokens, para evitar que eles sejam utilizados por um período prolongado.
* A rotação de tokens é importante para evitar que um token comprometido seja utilizado por um período prolongado.

**Armazenamento seguro (httpOnly cookies vs localStorage)**

* httpOnly cookies são mais seguros do que localStorage, pois não podem ser acessados por JavaScript, o que reduz o risco de ataques de XSS (Cross-Site Scripting).
* No entanto, é importante notar que httpOnly cookies podem ser vulneráveis a ataques de CSRF (Cross-Site Request Forgery).

**Implementação de refresh tokens**

* Refresh tokens são utilizados para obter um novo token de acesso quando o token atual expira.
* É importante implementar refresh tokens de forma segura, utilizando um algoritmo de assinatura seguro e definindo um tempo de expiração para os refresh tokens.

### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Use queries parametrizadas. Em vez de concatenar strings para criar queries, use parâmetros nomeados ou posicionais.

**Exemplo (Python com SQLite):**
```python
# Problema
query = "SELECT * FROM usuarios WHERE nome = '" + nome + "'"
cursor.execute(query)

# Solução
query = "SELECT * FROM usuarios WHERE nome = ?"
cursor.execute(query, (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações críticas, como atualizações de dados ou inserções, podem falhar parcialmente se não forem executadas dentro de uma transação.

**Solução:** Use transações para garantir a integridade dos dados.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
cursor.execute("INSERT INTO enderecos (rua, cidade) VALUES ('Rua 1', 'Cidade 1')")

# Solução
with connection:
    cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
    cursor.execute("INSERT INTO enderecos (rua, cidade) VALUES ('Rua 1', 'Cidade 1')")
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Consultas podem ser lentas se não houver índices nos campos de busca ou foreign keys.

**Solução:** Crie índices nos campos de busca e foreign keys.

**Exemplo (SQL):**
```sql
-- Problema
CREATE TABLE usuarios (

);

-- Solução

    id INTEGER PRIMARY KEY,
    nome VARCHAR(255),
    email VARCHAR(255)

CREATE INDEX idx_nome ON usuarios (nome);
CREATE INDEX idx_email ON usuarios (email);
```
