### 1. SQL Injection (queries concatenadas vs parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.
**Solução:** Use queries parametrizadas em vez de concatenadas.

Exemplo de código problemático:
```sql
query = "SELECT * FROM usuarios WHERE nome = '" + nome + "'"
```
Versão corrigida:
```sql
query = "SELECT * FROM usuarios WHERE nome = %s"
cursor.execute(query, (nome,))
```

### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como atualizações de dados, podem falhar parcialmente se não forem executadas em uma transação.
**Solução:** Use transações para garantir a integridade dos dados.

Exemplo de código problemático:
```sql
cursor.execute("UPDATE usuarios SET saldo = saldo - 10 WHERE id = 1")
cursor.execute("UPDATE contas SET saldo = saldo + 10 WHERE id = 2")
```
Versão corrigida:
```sql
with connection.cursor() as cursor:
    cursor.execute("BEGIN TRANSACTION")
    try:
        cursor.execute("UPDATE usuarios SET saldo = saldo - 10 WHERE id = 1")
        cursor.execute("UPDATE contas SET saldo = saldo + 10 WHERE id = 2")
        cursor.execute("COMMIT")
    except Exception as e:
        cursor.execute("ROLLBACK")
        raise e
```

### 3. Falta de índices em foreign keys e campos de busca

**Problema:** Consultas podem ser lentas se não houver índices nos campos de busca e foreign keys.
**Solução:** Crie índices nos campos de busca e foreign keys.

Exemplo de código problemático:
```sql
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(255),
    email VARCHAR(255)
)
```
Versão corrigida:
```sql
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(255) INDEX,
    email VARCHAR(255) UNIQUE INDEX
)
```

### 4. Uso de SELECT * desnecessário

**Problema:** O uso de `SELECT *` pode retornar mais dados do que necessário, aumentando o tempo de consulta e o uso de recursos.
**Solução:** Especifique os campos que você precisa em vez de usar `SELECT *`.

Exemplo de código problemático:
