**Algoritmos de assinatura JWT (HS256 vs RS256)**

* Recomendação: Utilizar RS256 (RSA com SHA-256) para assinatura de tokens JWT, pois é mais seguro do que HS256 (HMAC com SHA-256).
* Justificativa: RS256 utiliza uma chave pública e uma chave privada, o que torna mais difícil para um atacante obter a chave privada e falsificar tokens.

**Expiração e rotação de tokens**

* Recomendação: Utilizar tokens com expiração e rotação periódica.
* Justificativa: Isso ajuda a prevenir ataques de replay e a garantir que os tokens sejam atualizados regularmente.

**Armazenamento seguro (httpOnly cookies vs localStorage)**

* Recomendação: Utilizar httpOnly cookies para armazenar tokens de autenticação.
* Justificativa: httpOnly cookies são mais seguros do que localStorage, pois não podem ser acessados por JavaScript e são mais difíceis de serem roubados por ataques de cross-site scripting (XSS).

**Implementação de refresh tokens**

* Recomendação: Utilizar refresh tokens para atualizar tokens de acesso expirados.
* Justificativa: Isso ajuda a prevenir que os usuários precisem se autenticar novamente após a expiração do token de acesso.

**Rate limiting em endpoints de auth**

* Recomendação: Implementar rate limiting em endpoints de autenticação para prevenir ataques de brute force.
* Justificativa: Isso ajuda a prevenir que um atacante faça muitas tentativas de autenticação em um curto período de tempo.

```sql
-- Solução
```


### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como atualizações de dados, podem falhar parcialmente se não forem executadas dentro de uma transação.

```sql
-- Problema
UPDATE usuarios SET saldo = saldo - 100 WHERE id = 1;
UPDATE contas SET saldo = saldo + 100 WHERE id = 2;

-- Solução
BEGIN TRANSACTION;
UPDATE usuarios SET saldo = saldo - 100 WHERE id = 1;
UPDATE contas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;
```
