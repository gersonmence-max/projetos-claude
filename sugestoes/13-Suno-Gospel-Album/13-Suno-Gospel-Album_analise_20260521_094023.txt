### 1. SQL Injection (queries concatenadas vs parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas, onde os parâmetros são passados separadamente da query.

**Exemplo:**
```sql
-- Problema
SELECT * FROM usuarios WHERE nome = '" + nome + "'";

-- Solução
SELECT * FROM usuarios WHERE nome = @nome;
```

### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como atualizações de dados, podem ser comprometidas se não forem executadas dentro de uma transação.

**Solução:** Utilizar transações para garantir a integridade dos dados.

**Exemplo:**
```sql
-- Problema
UPDATE usuarios SET saldo = saldo - 100 WHERE id = 1;
UPDATE contas SET saldo = saldo + 100 WHERE id = 2;

-- Solução
BEGIN TRANSACTION;
UPDATE usuarios SET saldo = saldo - 100 WHERE id = 1;
UPDATE contas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;
```

### 3. Falta de índices em foreign keys e campos de busca

**Problema:** A falta de índices pode afetar o desempenho das consultas.

**Solução:** Criar índices nos campos de busca e foreign keys.

**Exemplo:**
```sql
-- Problema
CREATE TABLE usuarios (
  id INT PRIMARY KEY,
  nome VARCHAR(255),
  email VARCHAR(255)
);

-- Solução
CREATE TABLE usuarios (
  id INT PRIMARY KEY,
  nome VARCHAR(255),
  email VARCHAR(255)
);
CREATE INDEX idx_nome ON usuarios (nome);
CREATE INDEX idx_email ON usuarios (email);
```

### 4. Uso de SELECT * desnecessário

**Problema:** O uso de SELECT * pode retornar mais dados do que necessário, afetando o desempenho.

**Exemplo:**
```sql
-- Problema
SELECT * FROM usuarios;

-- Solução
SELECT id, nome, email FROM usuarios;
```
