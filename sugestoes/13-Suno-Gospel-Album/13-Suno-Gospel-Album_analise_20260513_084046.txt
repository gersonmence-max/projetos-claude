### Algoritmos de Assinatura JWT (HS256 vs RS256)

- **HS256 (HMAC com SHA-256)**: Utiliza uma chave secreta compartilhada para assinar e verificar a assinatura do token. É mais rápido e simples, mas requer que a chave secreta seja compartilhada entre as partes.
- **RS256 (RSA com SHA-256)**: Utiliza um par de chaves (pública e privada) para assinar e verificar a assinatura do token. É mais seguro, pois a chave privada não precisa ser compartilhada, mas é mais lento e complexo.

### Expiração e Rotação de Tokens

- **Expiração**: Tokens JWT devem ter um tempo de expiração para evitar que sejam válidos indefinidamente.
- **Rotação**: As chaves utilizadas para assinar os tokens devem ser rotacionadas periodicamente para minimizar o impacto de uma chave comprometida.

### Armazenamento Seguro

- **httpOnly Cookies**: São cookies que não podem ser acessados por JavaScript, reduzindo o risco de ataques de XSS.
- **localStorage**: É um armazenamento local no navegador, mas não é seguro para armazenar tokens de autenticação, pois pode ser acessado por JavaScript.

### Implementação de Refresh Tokens

- **Refresh Tokens**: São tokens utilizados para obter novos tokens de acesso quando o token de acesso expira.

### Rate Limiting em Endpoints de Auth

- **Rate Limiting**: É uma técnica para limitar o número de requisições que podem ser feitas em um determinado período de tempo, ajudando a prevenir ataques de brute force.

### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas. Em vez de concatenar strings para formar uma query, use parâmetros nomeados ou posicionais.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("SELECT * FROM usuarios WHERE nome = '" + nome + "'")

# Solução
cursor.execute("SELECT * FROM usuarios WHERE nome = ?", (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações que envolvem múltiplas queries podem deixar o banco de dados em um estado inconsistente se alguma delas falhar.

**Solução:** Utilizar transações. Isso garante que todas as operações dentro de uma transação sejam executadas como uma unidade, ou nenhuma delas será.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
cursor.execute("INSERT INTO perfis (usuario_id, descricao) VALUES (1, 'Perfil de João')")

# Solução
with connection:
    cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
    cursor.execute("INSERT INTO perfis (usuario_id, descricao) VALUES (1, 'Perfil de João')")
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Queries que filtram ou ordenam por campos sem índice podem ser lentas.

**Solução:** Criar índices nos campos relevantes.

**Exemplo (SQL):**
```sql
-- Problema
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);

-- Solução
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);
CREATE INDEX idx_usuarios_nome ON usuarios (nome);
CREATE INDEX idx_usuarios_email ON usuarios (email);
```
