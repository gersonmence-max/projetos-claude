### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilize queries parametrizadas. Em vez de concatenar strings para criar uma query, use parâmetros nomeados ou posicionais.

**Exemplo (Python com SQLite):**
```python
# Problema
nome = "João"
query = "SELECT * FROM usuarios WHERE nome = '" + nome + "'"
cursor.execute(query)

# Solução
nome = "João"
query = "SELECT * FROM usuarios WHERE nome = ?"
cursor.execute(query, (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações que envolvem múltiplas queries podem deixar o banco de dados em um estado inconsistente se alguma delas falhar.

**Solução:** Utilize transações. Comece uma transação antes de executar as queries e commit ou rollback dependendo do resultado.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
cursor.execute("INSERT INTO perfis (usuario_id, descricao) VALUES (1, 'Perfil de João')")

# Solução
try:
    cursor.execute("BEGIN TRANSACTION")
    cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
    cursor.execute("INSERT INTO perfis (usuario_id, descricao) VALUES (1, 'Perfil de João')")
    cursor.execute("COMMIT")
except:
    cursor.execute("ROLLBACK")
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Queries que utilizam foreign keys ou campos de busca podem ser lentas sem índices.

**Solução:** Crie índices nos campos relevantes.

**Exemplo (SQL):**
```sql
-- Problema
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);

CREATE TABLE perfis (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    descricao TEXT
);

-- Solução
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);

CREATE TABLE perfis (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    descricao TEXT,
    FOREIGN KEY (usuario_id) REFERENCES usuarios (id)
);

CREATE INDEX idx_usuario_id ON perfis (usuario_id);
```
