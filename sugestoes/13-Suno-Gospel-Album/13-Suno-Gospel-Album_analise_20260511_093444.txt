### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas. Em vez de concatenar strings para criar queries, use parâmetros nomeados ou posicionais.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("SELECT * FROM usuarios WHERE nome = '" + nome + "'")

# Solução
cursor.execute("SELECT * FROM usuarios WHERE nome = ?", (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Falta de transações pode levar a inconsistências nos dados em caso de falhas.

**Solução:** Utilizar transações para garantir a integridade dos dados.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
cursor.execute("INSERT INTO perfis (usuario_id, perfil) VALUES (1, 'admin')")

# Solução
with conn:
    cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
    cursor.execute("INSERT INTO perfis (usuario_id, perfil) VALUES (1, 'admin')")
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Falta de índices pode levar a queries lentas.

**Solução:** Criar índices em foreign keys e campos de busca.

**Exemplo (SQL):**
```sql
-- Problema
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome VARCHAR(255),
    email VARCHAR(255)
);

CREATE TABLE perfis (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    perfil VARCHAR(255)
);

-- Solução
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome VARCHAR(255),
    email VARCHAR(255)
);

CREATE TABLE perfis (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    perfil VARCHAR(255),
    FOREIGN KEY (usuario_id) REFERENCES usuarios (id)
);

CREATE INDEX idx_usuario_id ON perfis (usuario_id);
CREATE INDEX idx_nome ON usuarios (nome);
```
