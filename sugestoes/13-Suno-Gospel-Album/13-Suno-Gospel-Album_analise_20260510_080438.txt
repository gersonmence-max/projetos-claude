### 1. SQL Injection (queries concatenadas vs parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas, onde os parâmetros são passados separadamente da query.

**Exemplo:**

**Query problemática:**
```sql
cursor.execute("SELECT * FROM usuarios WHERE nome = '" + nome + "'")
```

**Query corrigida:**
```sql
cursor.execute("SELECT * FROM usuarios WHERE nome = %s", (nome,))
```

### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como atualizações de dados, podem falhar parcialmente, deixando o banco de dados em um estado inconsistente.

**Solução:** Utilizar transactions para garantir que as operações sejam executadas de forma atômica.

**Exemplo:**

**Código problemático:**
```python
cursor.execute("UPDATE usuarios SET saldo = saldo - 10 WHERE id = 1")
cursor.execute("UPDATE usuarios SET saldo = saldo + 10 WHERE id = 2")
```

**Código corrigido:**
```python
with connection.cursor() as cursor:
    cursor.execute("BEGIN TRANSACTION")
    try:
        cursor.execute("UPDATE usuarios SET saldo = saldo - 10 WHERE id = 1")
        cursor.execute("UPDATE usuarios SET saldo = saldo + 10 WHERE id = 2")
        cursor.execute("COMMIT")
    except Exception as e:
        cursor.execute("ROLLBACK")
        raise e
```

### 3. Falta de índices em foreign keys e campos de busca

**Problema:** Consultas que utilizam foreign keys ou campos de busca podem ser lentas devido à falta de índices.

**Solução:** Criar índices nos campos relevantes.

**Exemplo:**

**Tabela problemática:**
```sql
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(255),
    email VARCHAR(255)
)
```

**Tabela corrigida:**
```sql
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(255) INDEX,
    email VARCHAR(255) UNIQUE INDEX
)
```
