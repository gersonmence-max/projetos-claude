### 1. SQL Injection (Queries Concatenadas vs Parametrizadas)

**Problema:** Queries concatenadas são vulneráveis a ataques de SQL Injection.

**Solução:** Use queries parametrizadas. Em vez de concatenar strings para criar uma query, use parâmetros nomeados ou posicionais.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("SELECT * FROM usuarios WHERE nome = '" + nome + "'")

# Solução
cursor.execute("SELECT * FROM usuarios WHERE nome = ?", (nome,))
```

### 2. Ausência de Transactions em Operações Críticas

**Problema:** Operações que envolvem múltiplas queries podem falhar parcialmente, deixando o banco de dados em um estado inconsistente.

**Solução:** Use transações. Comece uma transação antes de executar as queries e commit ou rollback dependendo do resultado.

**Exemplo (Python com SQLite):**
```python
# Problema
cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
cursor.execute("INSERT INTO enderecos (usuario_id, rua, cidade) VALUES (1, 'Rua Exemplo', 'Cidade Exemplo')")

# Solução
with conn:
    cursor.execute("INSERT INTO usuarios (nome, email) VALUES ('João', 'joao@example.com')")
    cursor.execute("INSERT INTO enderecos (usuario_id, rua, cidade) VALUES (1, 'Rua Exemplo', 'Cidade Exemplo')")
```

### 3. Falta de Índices em Foreign Keys e Campos de Busca

**Problema:** Queries que envolvem foreign keys ou campos de busca podem ser lentas devido à falta de índices.

**Solução:** Crie índices nos campos relevantes.

**Exemplo (SQL):**
```sql
-- Problema
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);

CREATE TABLE enderecos (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    rua TEXT,
    cidade TEXT
);

-- Solução
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    email TEXT
);

CREATE TABLE enderecos (
    id INTEGER PRIMARY KEY,
    usuario_id INTEGER,
    rua TEXT,
    cidade TEXT,
    FOREIGN KEY (usuario_id) REFERENCES usuarios (id)
);

CREATE INDEX idx_usuario_id ON enderecos (usuario_id);
```
