### 1. SQL Injection (queries concatenadas vs parametrizadas)

**Problema:** Queries concatenadas podem permitir ataques de SQL Injection.

**Solução:** Utilizar queries parametrizadas.

**Exemplo:**

```python
# Problema
query = "SELECT * FROM musicas WHERE nome = '" + nome + "'"
cursor.execute(query)

# Solução
query = "SELECT * FROM musicas WHERE nome = %s"
cursor.execute(query, (nome,))
```

### 2. Ausência de transactions em operações críticas

**Problema:** Operações críticas, como upload de música e distribuição, podem falhar parcialmente, deixando o sistema em um estado inconsistente.

**Solução:** Utilizar transactions para garantir que as operações sejam executadas de forma atômica.

**Exemplo:**

```python
# Problema
upload_musica()
distribuir_musica()

# Solução
try:
    cursor.execute("BEGIN TRANSACTION")
    upload_musica()
    distribuir_musica()
    cursor.execute("COMMIT")
except Exception as e:
    cursor.execute("ROLLBACK")
    raise e
```

### 3. Falta de índices em foreign keys e campos de busca

**Problema:** Consultas podem ser lentas devido à falta de índices.

**Solução:** Criar índices em foreign keys e campos de busca.

**Exemplo:**

```sql
CREATE INDEX idx_nome ON musicas (nome);
CREATE INDEX idx_genero ON musicas (genero);
```

**Problema:** Consultas podem retornar mais dados do que o necessário, afetando o desempenho.

**Solução:** Utilizar SELECT com os campos específicos necessários.

**Exemplo:**

```sql

# Solução
SELECT nome, genero, duracao FROM musicas
```
